# 多模态故障知识图谱抽取系统使用指南

## 概述

本系统扩展了原有的故障知识图谱抽取工具，增加了对多模态数据的支持。通过集成成熟的编码器模型，系统能够处理文本、图像、表格、PDF等多种格式的数据，并将其转换为统一的向量表示，供大模型进行知识抽取。

## 新增功能

### 1. 多模态编码器
- **图像编码**: 使用CLIP模型进行视觉特征提取
- **文本编码**: 使用BGE-M3模型进行语义向量化
- **表格编码**: 将表格转换为结构化描述后编码
- **PDF编码**: 提取PDF文本内容后进行编码

### 2. 支持的文件格式
- **文本文件**: `.txt`, `.md`, `.json`
- **图像文件**: `.png`, `.jpg`, `.jpeg`, `.bmp`, `.gif`
- **表格文件**: `.csv`, `.xlsx`, `.xls`
- **文档文件**: `.pdf`

## 安装依赖

```bash
pip install -r requirements.txt
```

新增的依赖包括：
- `transformers`: Hugging Face模型库
- `torch`: PyTorch深度学习框架
- `pandas`: 表格数据处理
- `openpyxl`: Excel文件支持
- `sentence-transformers`: 文本嵌入模型

## 使用方法

### 1. 基本使用（多模态模式）
```bash
python main.py
```
默认使用多模态编码模式，自动检测并处理所有支持的格式。

### 2. 指定文件处理
```bash
python main.py --files image001.jpg table001.csv report.pdf
```

### 3. 传统文本模式
```bash
python main.py --mode legacy
```
仅使用文本提取，不进行向量编码。

### 4. 禁用编码功能
```bash
python main.py --no-encoding
```
使用多模态预处理器但禁用向量编码，仅提取文本内容。

## 技术架构

### 编码器选择

#### 图像编码器：CLIP (OpenAI)
- **模型**: `openai/clip-vit-base-patch32`
- **特点**: 通用视觉编码器，通过图文对比学习实现强大的图文对齐能力
- **适用场景**: 图像理解、图文检索、多模态理解

#### 文本编码器：BGE-M3 (BAAI)
- **模型**: `BAAI/bge-m3`
- **特点**: 当前SOTA文本嵌入模型，支持多语言、长文本和多粒度检索
- **备用模型**: `sentence-transformers/all-MiniLM-L6-v2`

#### 表格编码策略
1. 将表格转换为结构化文本描述
2. 包含数据维度、列名、样本数据
3. 添加统计信息（数值列）
4. 使用文本编码器进行向量化

### 向量化流程

```
原始文件 → 文件类型检测 → 内容提取 → 向量编码 → 大模型抽取
    ↓           ↓           ↓          ↓          ↓
  图像文件    图像编码器    OCR提取   CLIP编码   知识抽取
  表格文件    表格编码器   结构描述   BGE-M3编码 知识抽取
  PDF文件    文档编码器   文本提取   BGE-M3编码 知识抽取
  文本文件    文本编码器   直接读取   BGE-M3编码 知识抽取
```

### 1. 硬件要求

- **内存**: 至少8GB RAM

## 故障排除

### 常见问题

1. **模型加载失败**
   - 检查网络连接
   - 确保有足够的磁盘空间
   - 尝试使用备用模型

2. **内存不足**
   - 减少批量处理文件数量
   - 使用CPU模式
   - 关闭多模态编码

3. **文件格式不支持**
   - 检查文件扩展名
   - 确保文件未被损坏
   - 尝试手动转换格式
